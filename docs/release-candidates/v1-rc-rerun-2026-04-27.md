## Findings

The new provider limiter changed runtime behavior without changing artifact output. The anime build
completed at the same 50-candidate scope without hitting AniList `429`.

The rerun clarified the previous 50-to-48 anime observation: `--limit 50` selects 50 manami source
candidates, but the current normalizer silently skips candidates it cannot safely publish. In this
slice, two candidates were skipped before the normalized seed:

| Candidate Offset | Title | Reason |
|---:|---|---|
| 16 | `"Hitori de Ikirare Sou" tte Sore tte Nee, Homete Iru no?` | no supported source URL; only `anisearch.com` |
| 25 | `"Oshi no Ko" Final Season` | missing `animeSeason.year` |

This does not block v1 readiness for the RC, but source adapters should expose rejected candidate
counts and reasons instead of relying on local investigation.
