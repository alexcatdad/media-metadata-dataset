## Observations

The first 50-row anime attempt hit AniList `429 Too Many Requests` during metadata enrichment. The
RC exposed that the AniList relation and metadata fetchers needed bounded retry/backoff handling.
After adding retry handling, the same 50-row anime scope completed.

The anime source build used `--limit 50`, which selects 50 manami candidates. The normalized anime
seed contained 48 publishable rows because the normalizer skips candidates with missing required
fields or unsupported source URLs. This should be made explicit in adapter output before growing the
anime RC size further, but it did not block release readiness because the compiled artifact still had
meaningful anime, TV, and movie coverage.
